# Documentation

Specifications and documentation for **Home Project Management**.

Start with the [Project Overview](./project-overview.md) for what this tool is,
who it is for, and the architecture mandate (DDD + Hexagonal).

## Tech Stack

- **Frontend** — Next.js
- **Backend** — .NET Core API
- **Database** — PostgreSQL
- **Orchestration** — .NET Aspire

## Structure

| Folder | Purpose |
| --- | --- |
| [`specifications/`](./specifications/) | Feature specs, requirements, and design proposals |
| [`architecture/`](./architecture/) | System architecture, diagrams, and technical decisions |
| [`guides/`](./guides/) | How-to guides for development, setup, and operations |

## Conventions

- One document per topic; use kebab-case filenames (e.g. `task-scheduling.md`).
- Start each spec with a short summary, status, and date.
- Link related documents to keep navigation easy.
